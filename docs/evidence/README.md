# Evidence

This directory is the operator-of-record's evidence vault for compliance
and audit artifacts that are NOT generated by the application — typically
screenshots, dashboard exports, or third-party reports.

Evidence files are committed to the repo so the audit history is
versioned alongside the code. **Do not** commit anything containing live
secrets (deployment keys, API tokens, customer PII) — redact before
saving.

## What lives here

### Backup configuration screenshots (Story 5.6, ADR-0017)

Filename convention: `backup-config-YYYY-MM-DD.png`

Captured during the **quarterly backup verification** procedure
documented in [`docs/runbook.md` § Database backups → Backup configuration
verification](../runbook.md#backup-configuration-verification-quarterly).
Each screenshot must show the Convex dashboard's backup-settings panel
with **Daily backups: enabled** and **Retention: ≥ 30 days** clearly
visible. Redact deployment URLs / identifiers that are not already public.

The actual quarterly screenshot is an **operational step the on-call
dev performs in the Convex dashboard** — it is added to this folder when
the verification runs, not when the application is built. The presence
of this README is the placeholder that documents the requirement; the
PNG files appear as evidence accumulates.

The corresponding row in [`docs/adr/0017-database-backups.md` § Verification
ledger](../adr/0017-database-backups.md) references the evidence path,
closing the loop between procedure (runbook) → decision record (ADR) →
evidence (this folder).

### Future evidence categories

As other stories introduce operational verification steps (e.g.
Story 5.7's annual archival readability check, Story 6.x's expense
approval audit), they may file their own evidence here. Each category
should add a subsection to this README documenting its filename
convention and acceptance criteria.

## What does NOT live here

- Application-generated artifacts (PDFs, CSV exports). Those live in
  Convex File Storage and are downloaded on demand from the relevant
  admin page.
- Audit logs. Those are in the `auditLog` Convex table; query via
  `convex/auditLogQueries.ts`.
- Secrets / credentials. Use the operator's secure document store
  (1Password, hardware vault, etc.) — never commit secrets to the repo.
